## 일반 문제

### 빌드 및 컴파일

#### 문제: 의존성 누락으로 빌드 실패

**증상**:
```
error CS0246: The type or namespace name 'Microsoft.CodeAnalysis' could not be found
```

**해결책**:
```bash
dotnet restore
dotnet build
```

#### 문제: .NET SDK를 찾을 수 없음

**증상**:
```
The .NET SDK could not be found
```

**해결책**:
1. .NET 9 SDK 설치: https://dotnet.microsoft.com/download/dotnet/9.0
2. 설치 확인:
```bash
dotnet --version
```

### AI 연결 문제

#### 문제: Ollama 연결 거부

**증상**:
```
Failed to connect to Ollama at http://localhost:11434
```

**해결책**:
```bash
# Ollama가 실행 중인지 확인
ollama list

# Ollama 시작
ollama serve

# 연결 테스트
curl http://localhost:11434/api/tags
```

#### 문제: 모델을 찾을 수 없음

**증상**:
```
model "qwen2.5:7b" not found
```

**해결책**:
```bash
# 필요한 모델 가져오기
ollama pull qwen2.5:7b

# 사용 가능한 모델 나열
ollama list
```

#### 문제: 백련 404 오류

**증상**:
```
HTTP 404: Model not found
```

**해결책**:
1. API 키가 올바른지 확인
2. 모델 이름이 백련 카탈로그와 일치하는지 확인
3. 지역 엔드포인트가 올바른지 확인
4. 계정에 모델 접근 권한이 있는지 확인

### 런타임 문제

#### 문제: 포트가 이미 사용 중

**증상**:
```
HttpListenerException: Address already in use
```

**해결책**:

**Windows**:
```bash
netstat -ano | findstr :8080
taskkill /PID <PID> /F
```

**Linux/Mac**:
```bash
lsof -ti:8080 | xargs kill -9
```

**또는 구성에서 포트 변경**.

#### 문제: 생명체가 시작되지 않음

**증상**:
- 생명체 상태가 "Error" 표시
- 로그에 초기화 실패 표시

**해결책**:
1. 영혼 파일이 존재하고 유효한지 확인
2. AI 클라이언트가 구성되었는지 확인
3. 구체적인 오류에 대한 로그 확인:
```bash
tail -f logs/*.log
```

#### 문제: 메모리 부족

**증상**:
```
OutOfMemoryException
```

**해결책**:
1. 힙 크기 증가:
```bash
dotnet run --server.gcHeapCount 4
```

2. 오래된 데이터 정리:
```bash
# 오래된 로그 아카이브
mv logs/ logs-archive/
mkdir logs

# 오래된 메모리 정리
# Web UI 통해: 메모리 관리 > 정리
```

### 권한 문제

#### 문제: 권한 거부

**증상**:
```
Permission denied: disk:write
```

**해결책**:
1. 현재 권한 확인:
```bash
curl http://localhost:8080/api/permissions
```

2. 권한 부여:
```bash
curl -X POST http://localhost:8080/api/permissions \
  -H "Content-Type: application/json" \
  -d '{
    "resource": "disk:write",
    "allowed": true,
    "duration": 3600
  }'
```

3. 또는 Web UI 사용: 권한 관리

#### 문제: 권한이 만료되지 않음

**증상**:
- 권한이 만료 시간 후에도 여전히 유효

**해결책**:
1. 시스템 시계 동기화 확인
2. `expiresAt` 필드가 올바르게 설정되었는지 확인
3. 권한 캐시 지우기

### Web UI 문제

#### 문제: Web UI에 접근할 수 없음

**증상**:
- 브라우저에 "Connection refused" 표시

**해결책**:
1. 서버가 실행 중인지 확인
2. 올바른 URL 확인: `http://localhost:8080`
3. 방화벽 설정 확인
4. 시작 오류에 대한 로그 확인

#### 문제: SSE가 작동하지 않음

**증상**:
- 실시간 업데이트가 나타나지 않음
- 채팅이 스트리밍되지 않음

**해결책**:
1. 브라우저가 SSE를 지원하는지 확인
2. SSE에 대한 프록시 버퍼링 비활성화
3. 네트워크 안정성 확인
4. 다른 브라우저 시도

#### 문제: UI가 깨져 보임

**증상**:
- 스타일이 잘못됨
- 레이아웃이 깨짐

**해결책**:
1. 브라우저 캐시 지우기
2. 다른 스킨 시도: 설정 > 스킨
3. 브라우저 콘솔에서 오류 확인
4. 브라우저 확장 비활성화

### 저장 문제

#### 문제: 데이터를 읽기/쓰기할 수 없음

**증상**:
```
IOException: Access denied
```

**해결책**:
1. 파일 권한 확인
2. 저장 경로가 존재하는지 확인
3. 디스크 공간 확인
4. 적절한 권한으로 실행

#### 문제: 데이터 손상

**증상**:
- JSON 구문 분석 오류
- 데이터 손실

**해결책**:
1. 백업에서 복원
2. 저장 무결성 확인:
```bash
# Web UI 통해: 시스템 > 저장 확인
```

3. 손상된 파일 수동으로 수정

### 도구 실행 문제

#### 문제: 도구를 찾을 수 없음

**증상**:
```
Tool "xyz" not found
```

**해결책**:
1. 도구 이름이 올바른지 확인
2. 도구가 Tools 디렉토리에 있는지 확인
3. 프로젝트 재빌드
4. 도구가 올바르게 구현되었는지 확인

#### 문제: 도구가 오류 반환

**증상**:
```
Tool execution failed: ...
```

**해결책**:
1. 도구 로그 확인
2. 입력 매개변수 확인
3. 독립적으로 도구 테스트
4. 권한 확인

### 작업 노트 문제

#### 문제: 작업 노트를 생성할 수 없음

**증상**:
```
Failed to create work note
```

**해결책**:
1. 생명체가 존재하고 실행 중인지 확인
2. 저장 경로에 쓰기 권한이 있는지 확인
3. 내용이 비어 있지 않은지 확인 (내용 필수)
4. 자세한 오류 정보에 대한 로그 확인

#### 문제: 노트 검색 결과 없음

**증상**:
- 검색 키워드가 빈 결과 반환
- 관련 노트가 있음이 확실함

**해결책**:
1. 키워드 철자가 올바른지 확인
2. 더 일반적인 키워드 사용 시도
3. 노트에 해당 키워드가 포함되어 있는지 확인 (대소문자 구분)
4. `max_results` 매개변수 값 증가

#### 문제: 노트 디렉토리 생성이 느림

**증상**:
- 디렉토리 생성 시 응답 시간 김
- 생명체에 많은 노트가 있음 (>1000 페이지)

**해결책**:
1. 모든 노트를 순회해야 하므로 정상 현상
2. 오래된 노트를 정기적으로 아카이브 고려
3. 디렉토리 탐색 대신 검색 기능 사용
4. 계획된 최적화: 디렉토리 캐싱 메커니즘 추가

### 지식 네트워크 문제

#### 문제: 지식 쿼리가 빈 결과 반환

**증상**:
```
No knowledge triples found
```

**해결책**:
1. 주어와 술어의 철자 확인
2. 지식이 네트워크에 추가되었는지 확인
3. 퍼지 매칭을 위한 검색 기능 사용:
```json
{
  "action": "search",
  "query": "키워드"
}
```

#### 문제: 지식 경로 찾기 실패

**증상**:
```
No path found between concepts
```

**해결책**:
1. 두 개념이 지식 네트워크에 존재하는지 확인
2. 연결 경로가 있는지 확인 (직접적 또는 간접적 관계가 없을 수 있음)
3. 더 많은 지식을 추가하여 연결 구축 시도
4. 경로 길이 제한 낮추기 (설정된 경우)

#### 문제: 지식 검증 실패

**증상**:
```
Knowledge validation failed
```

**해결책**:
1. 트리플 형식이 올바른지 확인 (주어, 술어, 목적어 필수)
2. 신뢰도가 0.0-1.0 범위 내에 있는지 확인
3. 중복된 트리플이 있는지 확인
4. 구체적인 문제를 이해하기 위해 검증 오류 세부정보 확인

#### 문제: 지식 네트워크 통계 정보가 부정확

**증상**:
- 통계 숫자가 예상과 다름
- 지식 추가 후 통계가 업데이트되지 않음

**해결책**:
1. 통계 정보 업데이트에 몇 초 걸릴 수 있음 (캐시)
2. 삭제 작업이 성공적으로 실행되지 않았는지 확인
3. 통계 강제 새로고침을 위해 애플리케이션 재시작
4. API를 통해 통계 정보 재쿼리

### 프로젝트 관리 문제

#### 문제: 프로젝트를 생성할 수 없음

**증상**:
```
Failed to create project
```

**해결책**:
1. 프로젝트 이름이 비어 있지 않은지 확인 (필수)
2. 프로젝트 이름이 중복되지 않는지 확인
3. 저장 경로에 쓰기 권한이 있는지 확인
4. 자세한 오류 정보에 대한 로그 확인

#### 문제: 프로젝트 데이터 손실

**증상**:
- 프로젝트 정보를 로드할 수 없음
- 프로젝트 파일이 손상됨

**해결책**:
1. 프로젝트 저장 디렉토리가 존재하는지 확인
2. 백업에서 프로젝트 데이터 복원
3. JSON 파일 형식이 올바른지 확인
4. 손상된 프로젝트 파일 수동으로 수정

## 디버깅

### 상세 로그 활성화

구성 편집:
```json
{
  "logging": {
    "level": "debug"
  }
}
```

### 로그 확인

로그 저장 위치:
```
logs/
├── system.log
├── ai.log
├── permission.log
└── error.log
```

실시간 보기:
```bash
tail -f logs/*.log
```

### 디버거 사용

```bash
# 디버거로 실행
dotnet run --project src/SiliconLife.Default --configuration Debug

# 디버거 연결
# IDE 통해: 프로세스에 연결 > SiliconLife.Default
```

## 성능 문제

### 응답 시간 느림

**최적화**:
1. AI 모델 복잡도 낮추기
2. 캐싱 활성화

**모니터링**:
```bash
# Web UI 통해: 대시보드 > 메모리
```

**최적화**:
- 오래된 메모리 정리
- 컨텍스트 크기 줄이기
- 페이지네이션 구현

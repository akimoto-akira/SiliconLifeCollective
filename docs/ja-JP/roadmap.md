[中文](../zh-CN/roadmap.md) | [繁體中文](../zh-HK/roadmap.md) | [English](../roadmap.md)

## 基本原則

各フェーズは**実行可能で観測可能**なシステムで終了します。どのフェーズも「見せるものがないインフラの山」を生成することはありません。

## ~~フェーズ6: ルールに従う(パーミッションシステム)~~ ✅ 完了

**目標**: シリコンビーイングは承認なしに機密リソースにアクセス不可。

| # | モジュール | 説明 |
|---|--------|-------------|
| 6.1 | PermissionManager | ビーイングごとのプライベートインスタンス。コールバックベース、3方向結果(Allowed/Deny/AskUser)。クエリ優先度: HighDeny → HighAllow → Callback。IsCuratorフラグ |
| 6.2 | PermissionType列挙型 | NetworkAccess、CommandLine、FileAccess、Function、DataAccess |
| 6.3 | DefaultPermissionCallback | ネットワークホワイトリスト/ブラックリスト、CLI分類、ファイルパスセキュリティルール |
| 6.4 | GlobalACL | プレフィックスマッチングルールテーブル、ストレージに永続化 |
| 6.5 | UserFrequencyCache | HighAllow/HighDenyリスト。ユーザーオプトイン(自動検出ではない)。プレフィックスマッチング、メモリのみ、有効期限設定可能 |
| 6.6 | UserAskMechanism(コンソール) | AskUserが返されたときのy/nのコンソールプロンプト |
| 6.7 | エグゼキューターパーミッション統合 | すべてのエグゼキューターは実行前にパーミッションをチェック |
| 6.8 | IStorage分離注記 | IStorageはシステム内部永続化 — 直接ファイルアクセス、エグゼキューターを介して**ルーティングされない**、AIによって**制御不可**。エグゼキューターはAIツールによって開始されたIOのみを管理 |
| 6.9 | 監査ログ | タイムスタンプ、要求者、リソース、結果付きすべてのパーミッション決定をログ |

**成果物**: ビーイングが機密操作を試みたときにパーミッションプロンプトが表示。

**検証**: ビーイングにファイル削除を指示 → コンソールにパーミッションプロンプトが表示 → `n`を入力 → 操作拒否。ビーイングにホワイトリストサイトにアクセスするよう指示 → 直ちに許可。

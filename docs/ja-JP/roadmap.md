## ~~フェーズ 6：ルールに従う（権限システム）~~ ✅ 完了

**目標**：シリコンビーイングは承認なしに機密リソースにアクセスできません。

| # | モジュール | 説明 |
|---|--------|-------------|
| 6.1 | PermissionManager | 生命体ごとのプライベートインスタンス。コールバックベース、3 項結果（Allowed/Deny/AskUser）。クエリ優先度：HighDeny → HighAllow → コールバック。IsCurator フラグ |
| 6.2 | PermissionType 列挙型 | NetworkAccess、CommandLine、FileAccess、Function、DataAccess |
| 6.3 | DefaultPermissionCallback | ネットワークホワイトリスト/ブラックリスト、CLI 分類、ファイルパスセキュリティルール |
| 6.4 | GlobalACL | プレフィックスマッチングルールテーブル、Storage に永続化 |
| 6.5 | UserFrequencyCache | HighAllow/HighDeny リスト。ユーザーオプトイン（自動検出ではない）。プレフィックスマッチング、メモリのみ、有効期限設定可能 |
| 6.6 | ユーザー確認メカニズム（コンソール） | AskUser が返された場合、コンソールプロンプトで y/n |
| 6.7 | エグゼキューター権限統合 | すべてのエグゼキューターは実行前に権限をチェック |
| 6.8 | IStorage 分離注記 | IStorage はシステム内部永続化です — 直接ファイルアクセス、エグゼキューターを介して**ルーティングされない**、AI によって**制御されない**。エグゼキューターは AI ツールによって開始された IO のみを管理 |
| 6.9 | 監査ログ | タイムスタンプ、要求者、リソース、結果ですべての権限決定をログに記録 |

**成果物**：生命体が機密操作を試みると、権限プロンプトが表示されます。

**検証**：生命体にファイルの削除を指示 → コンソールに権限プロンプトが表示 → `n` と入力 → 操作が拒否。生命体にホワイトリストサイトにアクセスするよう指示 → 即座に許可。
